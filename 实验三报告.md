# 实验三：1.Project Euler 182 - RSA 加密指数选择

## 题目描述

本题给定两个素数 `p = 1009`、`q = 3643`，要求在 RSA 模数 `n = pq` 下寻找所有合法加密指数 `e` 中，使“未被隐藏的消息”数量最少的那些 `e`，并求这些 `e` 的总和。我的理解是，未被隐藏的消息指加密后仍然等于自身的消息，即 `m^e ≡ m (mod n)`。这类消息越多，说明该指数越容易让明文结构保留下来，因此题目本质是在分析 RSA 指数选择对固定点数量的影响。

## 过程

RSA 中合法的加密指数需要满足 `gcd(e, phi(n)) = 1`，其中 `phi(n) = (p - 1)(q - 1)`。对任意素数 `p`，方程 `m^e ≡ m (mod p)` 可以写成 `m(m^(e-1)-1) ≡ 0 (mod p)`。因此除了 `m = 0` 之外，还需要统计模 `p` 乘法群中满足 `m^(e-1) = 1` 的元素个数。由于模 `p` 的非零元素构成阶为 `p - 1` 的循环群，解的个数是 `gcd(e - 1, p - 1)`，再加上零元，一共有 `gcd(e - 1, p - 1) + 1` 个。

对 `q` 同理，再由中国剩余定理组合，模 `n` 下未隐藏消息数量为：

```text
(gcd(e - 1, p - 1) + 1) * (gcd(e - 1, q - 1) + 1)
```

程序枚举 `2 <= e < phi(n)`，跳过不满足互素条件的指数，然后计算上述数量，维护最小值和对应指数总和。核心代码如下：

```python
def count_unconcealed_messages(e: int, p: int = P, q: int = Q) -> int:
    return (gcd(e - 1, p - 1) + 1) * (gcd(e - 1, q - 1) + 1)
```

运行 `3.（1）project_euler_182.py` 后输出：

```text
minimum unconcealed messages: 9
sum of e values: 399788195976
pass
```

## 总结

这题让我更清楚地看到 RSA 中指数不是随便选的。`gcd(e, phi(n)) = 1` 只保证可以解密，但不同指数会产生不同数量的固定点。通过把问题拆到模 `p` 和模 `q` 上，再用循环群和中国剩余定理合并，可以避免直接枚举所有消息 `m`，把原本很大的问题转化为对指数的简单数论计算。

# 实验三：2.Implement RSA

## 题目描述

本题要求实现一个基础 RSA 加密系统，包括根据两个素数生成公钥和私钥、把消息整数加密为密文、再用私钥恢复明文。我的理解是，这道题重点不是实现复杂填充协议，而是把 RSA 的核心数学流程写清楚：选取 `p`、`q`，计算 `n` 和 `phi(n)`，选择与 `phi(n)` 互素的 `e`，再求 `d = e^(-1) mod phi(n)`。

## 过程

RSA 的密钥生成过程为：

```text
n = p * q
phi(n) = (p - 1)(q - 1)
ed ≡ 1 (mod phi(n))
public key = (e, n)
private key = (d, n)
```

加密时计算 `c = m^e mod n`，解密时计算 `m = c^d mod n`。由于 `ed = 1 + k phi(n)`，在满足 RSA 条件时，解密指数会把加密指数抵消，从而恢复消息代表元。

程序中使用扩展欧几里得算法求模逆：

```python
def invmod(a: int, modulus: int) -> int:
    g, x, _ = egcd(a % modulus, modulus)
    if g != 1:
        raise ValueError("value is not invertible")
    return x % modulus
```

再用 Python 内置的三参数 `pow(message, e, n)` 做快速模幂。运行 `3.（2）implement_rsa.py` 后输出：

```text
recovered: RSA demo
pass
```

## 总结

实现 RSA 时最容易忽视的是消息必须先转成小于 `n` 的整数，否则模运算会丢失信息。另一个关键点是 `e` 必须与 `phi(n)` 互素，否则不存在解密指数 `d`。本实验只实现了教材级 RSA，没有加入 OAEP 等安全填充，因此只能用于理解原理，不能直接当作实际安全加密方案。

# 实验三：3.RSA 加密体制破译

## 题目描述

本题来自 2016 全国高校密码数学挑战赛赛题三。题目给出一个 RSA 加密软件的加密案例和 Alice 被截获的多个加密帧。每个帧由 `1024 bit 模数 N | 1024 bit 加密指数 e | 1024 bit 密文 c` 三部分组成，明文分片最多 8 个字符，并在加密前填充成 512 bit：高 64 bit 是通信标志位，接着 32 bit 通信序号，中间补零，最后 64 bit 是明文分片 ASCII。我的理解是，本题不是靠暴力分解全部 1024 bit 模数，而是寻找参数选取不当造成的 RSA 弱点。

## 过程

程序先解析所有 `FrameXX` 文件，将每个 768 字节的 ASCII 十六进制串切成三段：`N`、`e`、`c`。随后使用多种攻击方法：

```python
Frame(
    name=path.name,
    n=int(data[:256], 16),
    e=int(data[256:512], 16),
    c=int(data[512:], 16),
)
```

第一类是共模攻击。`Frame0` 和 `Frame4` 使用了同一个模数 `N`，但加密指数不同且互素。如果两帧加密的是同一个明文，则由扩展欧几里得算法求出 `a e1 + b e2 = 1`，于是 `m = c1^a c2^b mod N`。程序恢复出序号 0 的分片 `My secre`。

第二类是共享素因子。对所有模数两两求 `gcd`，发现 `Frame1` 与 `Frame18` 共享同一个 512 bit 素因子。由 `p = gcd(N1, N2)` 和 `q = N / p` 可以计算 `phi(N)` 和私钥 `d`，从而解出序号 10、11 的分片 `m A to B` 与 `. Imagin`。

第三类是 Pollard p-1 分解。当某个素因子的 `p - 1` 比较光滑时，可以通过不断计算 `a^M mod N` 找到非平凡因子。程序分解了 `Frame2`、`Frame6`、`Frame19`。第四类是 Fermat 平方差分解，当 `p` 和 `q` 过于接近时，`N = a^2 - b^2`，可以从 `ceil(sqrt(N))` 开始搜索平方差，程序分解了 `Frame10`。

第五类是低指数广播攻击。`Frame3`、`Frame8`、`Frame12`、`Frame16`、`Frame20` 使用 `e = 5`，且加密了同一个明文分片。由于模数互素，可以先用中国剩余定理合并得到 `m^5`，再开整数五次方根，恢复序号 1 的分片 `t is a f`。

运行 `3.（3）rsa_challenge_three.py` 后能恢复的分片包括：

```text
seq=00 text='My secre'
seq=01 text='t is a f'
seq=10 text='m A to B'
seq=11 text='. Imagin'
```

程序同时说明，剩余帧没有发现共享因子、近素因子或足够数量的重复低指数样本，因此在不继续使用更高级的小根攻击或大整数分解的情况下不能完全恢复。

结合已恢复分片、通信序号和英文语义，可以进一步用猜测攻击补全原句。公开复现资料中给出的完整通关密语为：

```text
My secret is a famous saying of Albert Einstein. That is "Logic will get you from A to B. Imagination will take you everywhere."
```

## 总结

这题把 RSA 的多个经典风险集中到了一组数据里：同模数多指数会导致共模攻击，共享素因子会直接暴露私钥，`p - 1` 光滑会被 Pollard p-1 利用，`p` 和 `q` 太接近会被 Fermat 分解，低指数重复发送会触发广播攻击。我的主要收获是，RSA 的安全性不只依赖“1024 bit 模数看起来很大”，还强烈依赖参数生成质量和填充协议。实际应用中应使用安全随机数生成独立大素数，避免重复模数，使用标准公钥指数和 OAEP 这类随机填充，并逐步淘汰 1024 bit RSA。
